怎样发布一个 npm 包

vuePress-theme-reco 林琦彬    2020 - 2025

怎样发布一个 npm 包

Issho Lin 3/1/2020 组件

# 一、注册npm账号

点击此处前往npm官网注册 (opens new window)

注册后需要去邮箱中确认一下

点击邮件中的认证按钮，然后跟着提示一步步确认即可

# 二、电脑本地npm登录

  • 打开终端输入命令：npm adduser，依次输入刚刚注册 Username、Password 、Email，当终端输出如下图的信息时，说明已经登录成功

  • 查看 npm 登录信息，在终端输入npm config ls，在输出的信息中找到userconfig，复制后面的路径，然后在终端打开这个文件，就可以看到npm登录的信息

  • 退出 npm 的用户登陆：终端输入npm logout

# 三、创建npm包

这里采用Rollup来构建，仅仅只是因为小编一直用的webpack，还没用过Rollup，顺便学习一下；另一方面，毕竟社区流传着【webpack更适合应用层级的WebApp，而Rollup更适合使用在独立的JavaScript的模块库上】的结论，哈哈~

  1. 新建一个项目，并初始化 package.json
yarn init
  1. 安装Roolup
yarn add rollup --dev
  1. 项目配置

和webpack一样，es6转es5需要 babel 的支持，rollup 同样支持插件来实现代码压缩等

yarn add @babel/core @babel/preset-env -D
yarn add rollup-plugin-babel rollup-plugin-uglify -D

.babelrc：

{
    "presets": [
        "@babel/preset-env"
    ]
}

rollup.config.js：

import babel from 'rollup-plugin-babel'
import { uglify } from 'rollup-plugin-uglify'

export default {
    input: 'main.js',
    output: {
        file: './dist/bundle.js',
        format: 'cjs'
    },
    plugins: [
        babel({
            exclude: 'node_modules/**'
        }),
        uglify()
    ]
}

在 package.json 配置构建脚本

{
    "scripts": {
        "build": "npx rollup --config"
    }
}
  1. 撰写说明文档

README.md：

## 这是npm包的说明文档
  1. npm 包信息说明

package.json：

{
  "name": "", // 发布的npm包名字
  "version": "1.0.0", // npm包版本
  "description": "", // 包描述
  "main": "./dist/bundle.js", // 指定组件或函数的主入口文件
  "author": "",
  "license": "MIT",
  "repository": {
    "type": "git",
    "url": "https://github.com/Issho-lin/lunar-date.git"
  }
}
  1. 文件发布忽略声明

类似于.gitignore，声明发布npm包时需要忽略的文件，如果项目中没有.npmignore文件，但有.gitignore文件，则发布时会采用.gitignore的声明

.npmignore：

.*
*.md
node_modules/
rollup.config.js
src/

# 四、构建一个js工具函数包

  • 在入口文件export一个提供给用户使用的函数
import { GetLunarDay } from './src/tool'
/**
 * 获取当前农历日期
 * @param {Boolean} withYear 是否显示农历年
 */
export const lunarCalendarDate = withYear => {
    let D = new Date()
    let yy = D.getFullYear()
    let mm = D.getMonth() + 1
    let dd = D.getDate()
    if (yy < 100) yy = '19' + yy
    return GetLunarDay(yy, mm, dd, withYear)
}
  • 执行构建脚本
yarn build

# 五、发布 npm 包

  • 发布之前有几个注意点：
  1. 在发包之前，先到npm官网 (opens new window)搜索一下，避免有同名的包存在

  1. 一个版本只能发布一次，每一次迭代需要修改版本号
假设初始版本为1.0.0
➜ npm version preminor
v1.0.0-0
➜ npm version prepatch
v1.0.1-0
➜ npm version patch
v1.0.1
  1. 确保 npm 切换到官方镜像源
npm config get registry
npm config set registry https://registry.npmjs.org
  1. 确保本地电脑 npm 已登录
npm config ls
  • 确认没问题就执行 npm publish 正式发布
